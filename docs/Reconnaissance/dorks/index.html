Dorks are specialized search queries to uncover hidden, sensitive, or valuable information across Google, GitHub, Shodan, and Censys. This page focuses on crafting dorks for a single target domain like <domain>.

1. Google Dorks (Single Target)

Use advanced operators to find exposed files, panels, and leaks indexed for <domain>.

2. GitHub Dorks (Single Target)

Search public code for secrets, tokens, and config tied to <domain> (or orgs that own it).

3. Shodan Dorks (Single Target)

Query Internet-exposed services/devices linked to <domain> via banners or certificates.

4. Censys Dorks (Single Target)

Hunt hosts/services by TLS/HTTP metadata referencing <domain>.

Summary

  • Google finds indexed web content; GitHub spots code leaks; Shodan & Censys map exposed services.
  • Anchor each query to your <domain> (and optionally the owning <org>).
  • Use results to seed deeper recon (directory brute-force, JS analysis, parameter fuzzing).
  • Operate ethically and with permission.

Tip: Pair dorks with crawlers (katana/hakrawler) and URL probers (httpx) to validate findings at scale.

Tools in Kali Linux & BlackArch for Dorking/OSINT (Single Target)

Replace <domain> (and <org> when applicable). Some tools install via package manager or Python/pip.

Kali Linux (commonly available)

  • theHarvester ‚Äî Emails/hosts from engines/APIs for one domain.
    theharvester -d <domain> -b all -f harvester.html
  • googler ‚Äî Google from the terminal (craft Google dorks).
    googler --np --json 'site:<domain> ext:sql | ext:zip | ext:env'
  • shodan (CLI) ‚Äî Device/service search tied to the domain/cert.
    shodan search 'ssl:"<domain>"' --fields ip_str,port,hostnames,org
  • censys (CLI) ‚Äî Search hosts/services by TLS/HTTP metadata.
    censys search 'services.tls.certificates.leaf_data.names:"<domain>"'
  • github CLI (gh) ‚Äî Code search for domain strings/secrets.
    gh code search -q '"<domain>" (api_key OR token) filename:.env' --limit 100
  • trufflehog ‚Äî Secret detector for code dumps you pull.
    trufflehog filesystem ./repos_cloned_for_<domain>
  • gitleaks ‚Äî Fast rules-based secret scan on local repos/lists.
    gitleaks detect -s ./repos_cloned_for_<domain> -r gitleaks.json
  • waybackurls ‚Äî Archived URLs to pivot into dorks.
    echo <domain> | waybackurls > wayback.txt
  • gau ‚Äî Aggregate URLs (Wayback/CC/URLScan) for the domain.
    echo <domain> | gau > gau.txt
  • httpx ‚Äî Validate URL lists from dorks/archives (liveness/status).
    httpx -l gau.txt -mc 200,301,302 -o live.txt

BlackArch (recon/osint categories)

  • theHarvester ‚Äî One-domain OSINT aggregator (emails/hosts).
    theharvester -d <domain> -b all -f harvester.html
  • shodan (CLI) ‚Äî Banner and cert search linked to domain.
    shodan search 'hostname:"<domain>" OR ssl:"<domain>"'
  • censys (CLI) ‚Äî TLS/HTTP metadata queries for domain matches.
    censys search 'services.http.response.headers: "<domain>"'
  • github-dorkers (various) ‚Äî Dork GitHub for domain strings/secrets.
    gh code search -q '"<domain>" filename:.env OR filename:config.json'
  • trufflehog ‚Äî Secret scanner for cloned repos referencing domain.
    trufflehog filesystem ./repos_cloned_for_<domain>
  • gitleaks ‚Äî Ruleset-based scanner for token/secret patterns.
    gitleaks detect -s ./repos_cloned_for_<domain>
  • waybackurls ‚Äî Harvest URLs archived for the domain.
    echo <domain> | waybackurls > wayback.txt
  • gau ‚Äî Multi-source historical URLs for the domain.
    echo <domain> | gau > gau.txt
  • httpx ‚Äî Quickly verify/triage URLs from dork results.
    httpx -l wayback.txt -mc 200,301,302 -o live.txt
